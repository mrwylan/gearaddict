# Use Case: Remove Gear Item

## Overview

**Use Case ID:** UC-011
**Use Case Name:** Remove Gear Item
**Primary Actor:** Home Studio Owner
**Goal:** Remove a gear item from the personal inventory to reflect equipment no longer owned
**Status:** Approved
**Requirements:** FR-003

## Preconditions

- The user is logged in.
- The gear item to be removed belongs to the user's inventory.

## Main Success Scenario

1. User opens their gear inventory (UC-012) and selects the option to remove a gear item.
2. System prompts the user to confirm the removal.
3. User confirms the removal.
4. System removes the gear item from the user's inventory.
5. System redisplays the inventory without the removed item.

## Alternative Flows

### A1: User Cancels Removal

**Trigger:** User declines the confirmation prompt (step 3)
**Flow:**

1. System discards the removal request.
2. Use case ends with no change to the inventory.

## Postconditions

### Success Postconditions

- The gear item no longer appears in the user's inventory.
- The equipment's owner count on its catalog page is decremented by one (if the item was linked to a catalog entry).

### Failure Postconditions

- The gear item remains in the inventory.

## Business Rules

### BR-001: Confirmation Required

Removal requires explicit user confirmation to prevent accidental deletion.

### BR-002: Ownership Enforcement

A user may only remove gear items that belong to their own inventory.
