# Use Case: Add Gear Item

## Overview

**Use Case ID:** UC-009
**Use Case Name:** Add Gear Item
**Primary Actor:** Home Studio Owner
**Goal:** Add a piece of equipment to the personal gear inventory
**Status:** Approved
**Requirements:** FR-001, FR-005

## Preconditions

- The user is logged in.

## Main Success Scenario

1. User navigates to their gear inventory and selects "Add Gear Item".
2. System displays the add gear form with an equipment search field and a category selector.
3. User searches for the equipment using UC-006 (Search Equipment Catalog) and selects a match.
4. System pre-fills the equipment name and manufacturer from the catalog entry.
5. User selects a category (e.g. Synth, Effect, Keyboard, Interface).
6. User optionally adds a personal note or description for this item.
7. User submits the form.
8. System validates the submission.
9. System adds the gear item to the user's inventory.
10. System redisplays the inventory with the new item included.

## Alternative Flows

### A1: Equipment Not in Catalog

**Trigger:** User's search in step 3 yields no result for their specific device
**Flow:**

1. System offers the option to add a free-text device name.
2. User enters the manufacturer and device name manually.
3. Use case continues at step 5.

### A2: Category Not Selected

**Trigger:** User submits the form without selecting a category (step 7)
**Flow:**

1. System highlights the category field and prompts the user to select one.
2. User selects a category.
3. Use case continues at step 7.

### A3: Duplicate Gear Item

**Trigger:** The user already has the same equipment item in their inventory (step 8)
**Flow:**

1. System warns the user that the item is already in their inventory.
2. User confirms they wish to add it again (e.g. owning multiple units) or cancels.
3. If confirmed, use case continues at step 9. If cancelled, use case ends.

## Postconditions

### Success Postconditions

- The gear item appears in the user's inventory.
- The equipment's owner count on its catalog page is incremented by one.

### Failure Postconditions

- No gear item is added.
- The form is redisplayed with relevant error messages.

## Business Rules

### BR-001: Category Required

Every gear item must be assigned a category.

### BR-002: Equipment Link

Where possible, gear items should be linked to a catalog equipment entry to enable community features (owner count, discussions).
