# Use Case: Add Gear Item

## Overview

**Use Case ID:** UC-009
**Use Case Name:** Add Gear Item
**Primary Actor:** Home Studio Owner
**Goal:** Add a piece of equipment to the personal gear inventory
**Status:** Approved
**Requirements:** FR-001, FR-005

## Preconditions

- The user is logged in.

## Main Success Scenario

1. User navigates to their gear inventory and selects "Add Gear Item".
2. System displays the add gear form offering two entry modes: "Pick from catalog" and "Enter manually". The form shows a category selector in either mode.
3. User chooses "Pick from catalog" and searches for the equipment using UC-006 (Search Equipment Catalog), then selects a match.
4. System pre-fills the equipment name and manufacturer from the catalog entry as read-only references and links the item to the catalog entry.
5. User selects a category (e.g. Synth, Effect, Keyboard, Interface).
6. User optionally adds a personal note or description for this item.
7. User submits the form.
8. System validates the submission (see BR-003).
9. System adds the gear item to the user's inventory.
10. System redisplays the inventory with the new item included.

## Alternative Flows

### A1: Enter Equipment Manually (Free-Text)

**Trigger:** User chooses "Enter manually" in step 2, or the catalog search in step 3 yields no result for their specific device.
**Flow:**

1. System displays free-text fields for device name and manufacturer alongside the category selector.
2. User enters the manufacturer and device name manually.
3. User selects a category and optionally adds notes.
4. User submits the form.
5. System validates that both the manufacturer and device name are provided (see BR-004).
6. System stores the gear item with the free-text fields and no catalog link.
7. Use case continues at step 10.

### A2: Switch From Manual Entry to Catalog Pick

**Trigger:** While in manual entry mode, the user wants to attach a matching catalog entry instead.
**Flow:**

1. User selects "Pick from catalog" from the form.
2. System clears the free-text fields and shows the catalog search.
3. Use case continues at step 3.

### A3: Category Not Selected

**Trigger:** User submits the form without selecting a category (step 7)
**Flow:**

1. System highlights the category field and prompts the user to select one.
2. User selects a category.
3. Use case continues at step 7.

### A4: Duplicate Gear Item

**Trigger:** The user already has the same equipment item in their inventory (step 8)
**Flow:**

1. System warns the user that the item is already in their inventory.
2. User confirms they wish to add it again (e.g. owning multiple units) or cancels.
3. If confirmed, use case continues at step 9. If cancelled, use case ends.

## Postconditions

### Success Postconditions

- The gear item appears in the user's inventory.
- If the gear item is linked to a catalog entry, the equipment's owner count on its catalog page is incremented by one.
- If the gear item was entered as free-text, the inventory shows the user-supplied manufacturer and device name without a catalog link.

### Failure Postconditions

- No gear item is added.
- The form is redisplayed with relevant error messages.

## Business Rules

### BR-001: Category Required

Every gear item must be assigned a category.

### BR-002: Equipment Link

Where possible, gear items should be linked to a catalog equipment entry to enable community features (owner count, discussions). Free-text entries are permitted when the device is not in the catalog.

### BR-003: Catalog Pick Source

In catalog-pick mode, the equipment name and manufacturer are taken from the selected catalog entry and cannot be overridden by the user.

### BR-004: Free-Text Fields Required

In manual entry mode, both the manufacturer and the device name must be provided as non-empty free-text values.

### BR-005: Mutually Exclusive Modes

A gear item is either linked to a catalog entry (`equipment_id` set) or stored with free-text manufacturer and name. The two are mutually exclusive.
