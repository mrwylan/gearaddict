# Use Case: View Discussion Threads

## Overview

**Use Case ID:** UC-008
**Use Case Name:** View Discussion Threads
**Primary Actor:** Visitor
**Goal:** Read all community discussion threads for a specific piece of equipment
**Status:** Approved
**Requirements:** FR-011

## Preconditions

- The equipment item exists in the catalog.

## Main Success Scenario

1. Visitor navigates to the discussion threads section of an equipment page.
2. System displays a list of all discussion threads for the equipment, each showing the thread title, author username, creation date, and reply count.
3. Visitor selects a thread.
4. System displays the full thread: the opening post and all replies in chronological order, each showing author username, content, and timestamp.

## Alternative Flows

### A1: No Threads Exist

**Trigger:** No discussion threads have been started for this equipment (step 2)
**Flow:**

1. System displays an empty state message ("No discussions yet. Be the first to start one!").
2. An authenticated user may proceed to UC-015 (Start Discussion Thread).
3. Use case ends.

### A2: Thread Not Found

**Trigger:** The requested thread does not exist (step 3)
**Flow:**

1. System displays a "Thread not found" message.
2. Use case ends.

## Postconditions

### Success Postconditions

- The visitor has read the discussion thread content and replies.

### Failure Postconditions

- Not applicable; this is a read-only operation.

## Business Rules

### BR-001: Public Access

Discussion threads and their replies are publicly visible without authentication.

### BR-002: Chronological Order

Replies within a thread are displayed in ascending chronological order (oldest first).

### BR-003: Thread List Order

Threads in the list are ordered by the date of the most recent reply (most recently active first).
