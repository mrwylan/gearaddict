# Use Case: Reply to Discussion Thread

## Overview

**Use Case ID:** UC-016
**Use Case Name:** Reply to Discussion Thread
**Primary Actor:** Home Studio Owner
**Goal:** Add a reply to an existing discussion thread to engage with other users about a piece of gear
**Status:** Approved
**Requirements:** FR-010

## Preconditions

- The user is logged in.
- The discussion thread exists and is open for replies.

## Main Success Scenario

1. User views a discussion thread (UC-008) and selects "Reply".
2. System displays a reply form below the thread.
3. User writes the reply content.
4. User submits the reply.
5. System validates the submission (reply body is non-empty and within length limits).
6. System creates the reply attributed to the user and appended to the thread.
7. System redisplays the thread with the new reply visible at the bottom.

## Alternative Flows

### A1: Empty Reply

**Trigger:** User submits the form with an empty reply body (step 4)
**Flow:**

1. System highlights the reply field and displays a validation message.
2. User enters reply content.
3. Use case continues at step 4.

### A2: Reply Exceeds Maximum Length

**Trigger:** The reply body exceeds the allowed character limit (step 5)
**Flow:**

1. System highlights the reply field and displays the character limit.
2. User shortens the content.
3. Use case continues at step 4.

### A3: User Not Logged In

**Trigger:** An unauthenticated visitor attempts to reply
**Flow:**

1. System redirects the visitor to the log-in page.
2. After successful authentication, use case resumes at step 2.

## Postconditions

### Success Postconditions

- The reply is appended to the thread and immediately visible to all visitors.
- The thread's "last activity" timestamp is updated.

### Failure Postconditions

- No reply is added to the thread.
- The reply form is redisplayed with validation errors.

## Business Rules

### BR-001: Authentication Required

Only authenticated users may post replies.

### BR-002: Reply Body Length

Reply content must be between 1 and 10,000 characters.

### BR-003: Chronological Ordering

Replies are appended to the thread in chronological order and cannot be reordered.
