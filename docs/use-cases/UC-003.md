# Use Case: Log Out

## Overview

**Use Case ID:** UC-003
**Use Case Name:** Log Out
**Primary Actor:** Home Studio Owner
**Goal:** End the current authenticated session and return to a guest state
**Status:** Approved
**Requirements:** FR-015
**Navigation Entry:** Triggered from the user-avatar menu in the left navigation rail (see [requirements.md § Navigation Layout](../requirements.md#navigation-layout)); the post-logout landing renders outside the shell.
**Wireframe(s):** [Guest dashboard — Dashboard V04](../wireframes/screenshots/04-variant.png) (post-logout landing — see [Wireframes.html](../wireframes/Wireframes.html))

## Preconditions

- The user is currently logged in with an active session.

## Main Success Scenario

1. User selects the "Log Out" action from the navigation.
2. System invalidates the current session.
3. System redirects the user to the home page as an unauthenticated visitor.

## Alternative Flows

None.

## Postconditions

### Success Postconditions

- The user's session is invalidated and no longer usable.
- The user is redirected to the public home page.

### Failure Postconditions

- Not applicable; log out is a destructive-safe operation with no error states.

## Business Rules

### BR-001: Session Invalidation

The session token must be invalidated server-side upon log out, not merely removed from the client.
