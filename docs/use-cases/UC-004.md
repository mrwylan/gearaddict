# Use Case: Edit Profile

## Overview

**Use Case ID:** UC-004
**Use Case Name:** Edit Profile
**Primary Actor:** Home Studio Owner
**Goal:** Update personal profile information to personalize presence on the platform
**Status:** Approved
**Requirements:** FR-016
**Navigation Entry:** Reachable via the **User Avatar** entry at the bottom of the left navigation rail (see [requirements.md § Navigation Layout](../requirements.md#navigation-layout)).
**Wireframe(s):** [The Rack — Profile V01](../wireframes/screenshots/05-variant.png) (see [Wireframes.html](../wireframes/Wireframes.html))

## Preconditions

- The user is logged in.

## Main Success Scenario

1. User navigates to their profile settings page.
2. System displays the profile form pre-populated with the current profile information (display name, bio).
3. User modifies one or more fields.
4. User saves the changes.
5. System validates the submitted data.
6. System persists the updated profile information.
7. System displays a confirmation that the profile has been updated.

## Alternative Flows

### A1: Validation Error

**Trigger:** A submitted field violates a constraint (e.g. display name too long) (step 5)
**Flow:**

1. System highlights the offending field(s) and displays a descriptive error.
2. User corrects the value.
3. Use case continues at step 4.

### A2: User Cancels

**Trigger:** User discards changes before saving (step 3 or 4)
**Flow:**

1. System discards unsaved changes.
2. Use case ends with no modification to the profile.

## Postconditions

### Success Postconditions

- The profile information is updated and immediately reflected on the user's public profile page.

### Failure Postconditions

- The profile data is unchanged.
- The form is redisplayed with validation errors.

## Business Rules

### BR-001: Display Name Length

Display names must be between 1 and 50 characters.

### BR-002: Bio Length

The bio field must not exceed 500 characters.
