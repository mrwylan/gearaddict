# Use Case: Browse User Inventory

## Overview

**Use Case ID:** UC-013
**Use Case Name:** Browse User Inventory
**Primary Actor:** Home Studio Owner
**Goal:** Discover what equipment another user owns by viewing their public gear inventory
**Status:** Approved
**Requirements:** FR-013

## Preconditions

- The viewing user is logged in.
- The target user has a public gear inventory.

## Main Success Scenario

1. User navigates to another user's profile page (e.g. from a gear connection in UC-014 or a discussion thread in UC-008).
2. System displays the target user's public profile, including their gear inventory.
3. The inventory shows each gear item with device name, manufacturer, and category.
4. User browses the inventory.
5. User optionally selects a piece of equipment to view its equipment page (UC-007).

## Alternative Flows

### A1: Target User Has No Public Inventory

**Trigger:** The target user's inventory is empty or set to private (step 2)
**Flow:**

1. System displays the profile page without an inventory section, or with an empty state message.
2. Use case ends.

### A2: Profile Not Found

**Trigger:** The target user does not exist (step 1)
**Flow:**

1. System displays a "User not found" message.
2. Use case ends.

## Postconditions

### Success Postconditions

- The user has browsed the target user's public gear inventory.

### Failure Postconditions

- Not applicable; this is a read-only operation.

## Business Rules

### BR-001: Public Inventory Only

Only the public portion of another user's inventory is visible.

### BR-002: No Edit Access

The viewing user cannot modify another user's inventory.
