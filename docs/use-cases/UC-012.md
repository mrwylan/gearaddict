# Use Case: View Own Inventory

## Overview

**Use Case ID:** UC-012
**Use Case Name:** View Own Inventory
**Primary Actor:** Home Studio Owner
**Goal:** See an overview of all personally owned equipment at a glance
**Status:** Approved
**Requirements:** FR-004
**Navigation Entry:** Reachable via the **My Inventory** entry in the left navigation rail (see [requirements.md § Navigation Layout](../requirements.md#navigation-layout)).
**Wireframe(s):** [Studio at a glance — Dashboard V02](../wireframes/screenshots/02-variant.png) (see [Wireframes.html](../wireframes/Wireframes.html))

## Preconditions

- The user is logged in.

## Main Success Scenario

1. User navigates to their gear inventory page.
2. System displays all gear items in the user's inventory, each showing the device name, manufacturer, category, and personal note (if any).
3. User optionally filters the inventory by category.
4. System updates the displayed list to show only items matching the selected category.
5. User optionally selects a gear item to edit (UC-010) or remove (UC-011).

## Alternative Flows

### A1: Inventory Is Empty

**Trigger:** The user has no gear items in their inventory (step 2)
**Flow:**

1. System displays an empty state message and a prominent "Add Gear Item" call to action.
2. Use case continues at UC-009 if the user proceeds.

### A2: No Items Match Filter

**Trigger:** No gear items match the selected category filter (step 4)
**Flow:**

1. System displays an empty state message for the selected filter.
2. User selects a different category or clears the filter.
3. Use case continues at step 2.

## Postconditions

### Success Postconditions

- The user has an up-to-date overview of their equipment collection.

### Failure Postconditions

- Not applicable; this is a read-only view.

## Business Rules

### BR-001: Private by Default

A user's inventory is visible only to themselves unless they have made it public (see UC-013).

### BR-002: Default Sort Order

Gear items are displayed alphabetically by device name by default.

### BR-003: Infinite Scroll

The inventory list uses infinite scroll with lazy loading: an initial batch of gear items is fetched on page load, and additional batches are fetched as the user scrolls toward the end of the list. No page-number controls are shown.
