# Use Case: Curate Equipment Catalog

## Overview

**Use Case ID:** UC-017
**Use Case Name:** Curate Equipment Catalog
**Primary Actor:** Catalog Curator
**Goal:** Maintain the accuracy and completeness of the shared equipment catalog and its manufacturer list by adding, editing, or removing entries
**Status:** Draft
**Requirements:** FR-023

## Preconditions

- The curator is logged in.
- The curator has the catalog curation permission.

## Main Success Scenario

1. Curator navigates to the catalog curation area from the administration menu.
2. System displays the curation workspace with two sections: Equipment entries and Manufacturers.
3. Curator selects the Equipment section.
4. System displays the list of equipment entries with name, manufacturer, category, and number of linked gear items.
5. Curator selects "Add Equipment".
6. System displays the equipment entry form with fields for name, manufacturer (selected from the existing manufacturer list), category, and description.
7. Curator fills in the equipment details and submits the form.
8. System validates the submission against the catalog rules (see BR-001, BR-002, BR-003).
9. System stores the new equipment entry and adds it to the catalog.
10. System redisplays the equipment list with the new entry included and a confirmation message.

## Alternative Flows

### A1: Edit Existing Equipment Entry

**Trigger:** Curator selects an existing equipment entry from the list (step 4)
**Flow:**

1. Curator opens the entry and selects "Edit".
2. System displays the equipment form pre-populated with the existing values.
3. Curator updates one or more fields and submits.
4. System validates the changes (BR-001, BR-002, BR-003).
5. System persists the updated entry and refreshes the list.
6. Use case ends.

### A2: Remove Equipment Entry

**Trigger:** Curator selects an existing equipment entry and chooses "Remove" (step 4)
**Flow:**

1. System displays a confirmation dialog showing the number of gear items currently linked to this entry.
2. Curator confirms the removal.
3. If linked gear items exist (BR-004), system unlinks those gear items so they fall back to free-text equipment data.
4. System deletes the equipment entry from the catalog.
5. System redisplays the equipment list without the removed entry.
6. Use case ends.

### A3: Add Manufacturer

**Trigger:** Curator selects the Manufacturers section (step 2) and chooses "Add Manufacturer"
**Flow:**

1. System displays the manufacturer form with a name field.
2. Curator enters the manufacturer name and submits.
3. System validates uniqueness (BR-005) and minimum length (BR-006).
4. System stores the new manufacturer and redisplays the manufacturer list with the new entry.
5. Use case ends.

### A4: Edit Manufacturer

**Trigger:** Curator selects an existing manufacturer from the manufacturer list
**Flow:**

1. Curator opens the manufacturer and selects "Edit".
2. System displays the manufacturer form pre-populated with the current name.
3. Curator updates the name and submits.
4. System validates the change (BR-005, BR-006) and propagates the renamed manufacturer to all linked equipment entries.
5. System redisplays the manufacturer list with the updated name.
6. Use case ends.

### A5: Remove Manufacturer

**Trigger:** Curator selects "Remove" on a manufacturer entry
**Flow:**

1. System displays a confirmation dialog showing the number of equipment entries currently using this manufacturer.
2. If any equipment entries reference the manufacturer (BR-007), system blocks the removal and instructs the curator to reassign or remove those equipment entries first.
3. If no entries reference it, the curator confirms and the system removes the manufacturer.
4. Use case ends.

### A6: Validation Failure on Equipment Entry

**Trigger:** Required fields are missing or invalid on submit (step 8)
**Flow:**

1. System highlights the offending fields and displays validation messages.
2. Curator corrects the input.
3. Use case continues at step 7.

### A7: Duplicate Equipment Entry

**Trigger:** The combination of name and manufacturer already exists (step 8, see BR-002)
**Flow:**

1. System displays an error indicating the duplicate entry and offers to open the existing record.
2. Curator either edits the existing entry (continues at A1 step 2) or cancels.
3. Use case ends.

### A8: Permission Missing

**Trigger:** A logged-in user without the curation permission attempts to access the curation workspace (step 1)
**Flow:**

1. System denies access and displays an authorization error.
2. Use case ends.

## Postconditions

### Success Postconditions

- The catalog reflects the curator's changes (additions, edits, or removals) for equipment entries and the manufacturer list.
- Equipment entries that were modified or removed propagate consistently to dependent views (browse, search, equipment page, gear inventory).
- An audit trail of the curation action is recorded with the curator's identity and timestamp.

### Failure Postconditions

- The catalog state is unchanged.
- The system displays a validation or authorization error to the curator.

## Business Rules

### BR-001: Required Equipment Fields

Every equipment entry must have a non-empty name, an assigned manufacturer, and a category.

### BR-002: Equipment Uniqueness

The combination of name and manufacturer must be unique across the catalog (consistent with the EQUIPMENT entity constraint).

### BR-003: Allowed Categories

The category must be one of the supported values (Synth, Effect, Keyboard, Interface, Other).

### BR-004: Removal Cascading

Removing an equipment entry that is referenced by user gear items must first unlink those items so the inventory data is preserved as free-text entries.

### BR-005: Manufacturer Uniqueness

Manufacturer names must be unique (case-insensitive).

### BR-006: Manufacturer Name Length

Manufacturer names must be between 2 and 100 characters.

### BR-007: Manufacturer Removal Blocked While Referenced

A manufacturer cannot be removed while any equipment entry references it. The curator must reassign or remove those equipment entries first.

### BR-008: Audit Trail

Every curation action (create, update, delete on equipment or manufacturer) must be recorded with the curator's user id and a timestamp.

### BR-009: Curation Permission

Only users granted the catalog curation permission may perform this use case. Regular Home Studio Owners and Visitors are excluded.
