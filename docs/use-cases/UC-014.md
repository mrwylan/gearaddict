# Use Case: Discover Gear Connections

## Overview

**Use Case ID:** UC-014
**Use Case Name:** Discover Gear Connections
**Primary Actor:** Home Studio Owner
**Goal:** Find other users who own the same gear to connect with like-minded musicians
**Status:** Approved
**Requirements:** FR-012
**Navigation Entry:** Reachable via the **Gear Connections** entry in the left navigation rail (see [requirements.md § Navigation Layout](../requirements.md#navigation-layout)).

## Preconditions

- The user is logged in.
- The user has at least one gear item in their inventory that is linked to a catalog equipment entry.

## Main Success Scenario

1. User navigates to their gear connections page or opens a specific gear item.
2. System displays a list of other users who own the same equipment, showing their username and the shared gear item(s).
3. User selects another user from the list.
4. System navigates to that user's public profile and inventory (UC-013).

## Alternative Flows

### A1: No Connections Found for a Gear Item

**Trigger:** No other users own the same equipment as the selected item (step 2)
**Flow:**

1. System displays a message indicating no connections yet for this item.
2. Use case ends.

### A2: Inventory Has No Catalog-Linked Items

**Trigger:** All of the user's gear items are free-text entries not linked to the catalog (step 1)
**Flow:**

1. System informs the user that connections are available only for catalog-linked items.
2. Use case ends.

## Postconditions

### Success Postconditions

- The user is aware of other community members who share the same gear and can visit their profiles.

### Failure Postconditions

- Not applicable; this is a read-only discovery operation.

## Business Rules

### BR-001: Catalog Link Required

Gear connections are only available for items linked to a catalog equipment entry, not for free-text items.

### BR-002: Exclude Self

The user's own account is excluded from the connections list.

### BR-003: Public Inventory Required

Only users with a public inventory appear in the connections list.

### BR-004: Infinite Scroll

The connections list uses infinite scroll with lazy loading: an initial batch of matching users is fetched on page load, and additional batches are fetched as the user scrolls toward the end of the list. No page-number controls are shown.
