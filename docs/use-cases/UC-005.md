# Use Case: Browse Equipment Catalog

## Overview

**Use Case ID:** UC-005
**Use Case Name:** Browse Equipment Catalog
**Primary Actor:** Visitor
**Goal:** Discover gear used by the community by browsing the equipment catalog
**Status:** Approved
**Requirements:** FR-008

## Preconditions

- The equipment catalog contains at least one entry.

## Main Success Scenario

1. Visitor navigates to the equipment catalog page.
2. System displays a lazily-loaded list of equipment items (see BR-002), each showing the device name, manufacturer, and category.
3. Visitor optionally filters the list by category.
4. System updates the displayed list to match the selected filter.
5. Visitor selects a piece of equipment from the list.
6. System navigates to the equipment page for the selected item (UC-007).

## Alternative Flows

### A1: Catalog Is Empty

**Trigger:** No equipment items exist in the catalog (step 2)
**Flow:**

1. System displays an empty state message ("No equipment in the catalog yet").
2. Use case ends.

### A2: No Results for Selected Filter

**Trigger:** No items match the selected category filter (step 4)
**Flow:**

1. System displays an empty state message for the selected filter.
2. Visitor selects a different category or clears the filter.
3. Use case continues at step 2.

## Postconditions

### Success Postconditions

- The visitor has an overview of available equipment and can navigate to individual equipment pages.

### Failure Postconditions

- Not applicable; this is a read-only browsing operation.

## Business Rules

### BR-001: Default Sort Order

Equipment items are displayed alphabetically by manufacturer and device name by default.

### BR-002: Infinite Scroll

The catalog uses infinite scroll with lazy loading: an initial batch of items is fetched on page load, and additional batches are fetched automatically as the visitor scrolls toward the end of the list. This avoids loading excessive data in a single request while keeping navigation continuous (no page-number controls).
